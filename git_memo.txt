1.gitの初期化
git init

2.ファイルをステージングエリアに変更を追加
git add ファイル
全ファイルを追加
git add . 

削除の場合
git rm ファイル

3.ローカルリポジトリにファイルを登録
git commit -v
-v:変更内容の出力

※コメントはデフォルトだとbimエディタで入力することになる
【コミットメッセージの書き方】
1行目：変更内容の要約
2行目：空行
3行目：変更した理由

4.リモートリポジトリのステージングエリアに登録
git remote add origin リモートリポジトリのアクセス先

5.リモートリポジトリに登録
git push -u origin master

備考
・変更履歴の確認
git log
--oneline：1行で表示する
-p ファイル：変更の差分表示 jキー：↓、kキー：↑
-n 数字：最新からのコミットを表示


・現在の状態の確認
git status

・変更差分を確認
git diff

【add, commitする前に確認】
ステージとの差分
git diff

ステージとコミットの差分
git diff HEAD
